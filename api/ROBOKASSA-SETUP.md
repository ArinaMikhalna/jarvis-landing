# Подключение Robokassa — что осталось сделать

Фронт готов: кнопки тарифов ведут на `/api/pay?plan=baza` и `/api/pay?plan=pro`.
Функция `api/pay.js` считает подпись на сервере и редиректит на оплату Robokassa.
Секреты берутся ТОЛЬКО из переменных окружения — в коде их нет.

## 1. Задеплоить сайт с поддержкой функций
Локальный сервер (serve-jarvis.js) функции `/api/*` не выполняет — оплата заработает после деплоя.
Проще всего: **Vercel** (или Netlify). Папка `api/` автоматически становится serverless-функциями.

## 2. Прописать переменные окружения (в кабинете хостинга, не в коде!)
- `ROBO_LOGIN` — MerchantLogin (идентификатор магазина из кабинета Robokassa)
- `ROBO_PASS1` — Пароль №1 (секретный)
- `ROBO_TEST` — `1` для тестового режима, убрать/`0` для боевого

## 3. Настроить в кабинете Robokassa
- **Success URL** — страница «спасибо за оплату» (например `/success.html`)
- **Fail URL** — страница «оплата не прошла»
- **Result URL** — серверный коллбэк подтверждения оплаты (рекомендуется, для надёжности).
  Подпись результата проверяется как `md5(OutSum:InvId:Пароль№2)`. Если нужно — допишу `api/result.js`.

## 4. Фискализация (54-ФЗ) — проверить!
Как ИП ты, скорее всего, обязана выдавать чек. В Robokassa включается фискализация
(передача состава заказа, параметр Receipt). Уточни в поддержке Robokassa и в своей бухгалтерии —
если нужно, добавлю формирование чека в `api/pay.js`.

## Что дать мне, чтобы доделать (если нужен Result-коллбэк/чеки)
- НЕ присылай пароли в чат. Их ставишь сама в переменные окружения на хостинге.
- Скажи: нужен ли Result URL (подтверждение) и фискализация — тогда допишу серверную часть.
